JobPilot：从投简历到造工具，我用 Electron 写了一个求职自动化助手

2026-06-19

秋招不是体力活，应该是智力活。与其在几十个标签页之间反复横跳，不如让工具替你跑腿。

背景：秋招季的焦虑与信息洪流

我是姚军，数字经济专业2027届硕士。和大多数同学一样，秋招和实习季是我压力最大的阶段。

招聘信息散布在微信群、公众号、各公司官网、BOSS直聘、牛客网……每天光是刷消息、筛岗位、复制链接、打开官网、注册账号、填一遍又一遍重复的个人信息，就能耗掉大半天。投一份简历本身不难，但当你要同时投递二三十家公司时，"填表"这件事就变成了纯粹的时间黑洞。

我开始想：这个过程能不能自动化？

灵感时刻：一次成功的投递

转折点出现在一次实习申请。某金融数据公司的实习岗位需要在官网填写一系列表单。我尝试用浏览器自动化工具（Playwright）写了一个脚本，自动填完了姓名、学校、专业、实习时间等字段——然后成功提交了。

那一刻我意识到：如果我能把这个"自动填表"的能力封装成一个产品，让它不仅能填一家公司，而是能解析各种渠道的招聘信息、管理投递状态、自动完成表单，那将极大缩短求职的时间成本。

于是 JobPilot 诞生了。

JobPilot 能做什么

JobPilot 的核心功能可以概括为三个阶段：

1. 信息采集：从杂乱消息中提取结构化数据

微信群里的一条消息可能是这样的：

"【实习招聘】XX科技-电商部门招前端实习生，base北京，要求2026届及以上，有意者发简历到 xxx@example.com，内推码：ABC123"

JobPilot 能从中智能解析出：

  • 公司：XX科技
  • 部门：电商
  • 岗位：前端实习生
  • 工作地点：北京
  • 联系方式：xxx@example.com
  • 内推码：ABC123

2. 投递管理：看板式的状态追踪

每一条投递都有一个状态，从 待处理 → 填写中 → 已投递 → 面试 → Offer → 已拒绝，一目了然。再也不用打开 Excel 或者靠记忆去追踪"这家我投了没有"。

3. 自动填表：Playwright 驱动浏览器

选一个职位，点击"自动投递"，JobPilot 会打开浏览器、导航到招聘页面、自动填写表单字段并提交。整个过程模拟真人操作，带有随机延迟，不会被识别为机器人。

技术选型：为什么是这些技术

技术选择理由
Electron求职工具需要运行在桌面端，用户需要在本地浏览器上操作。Electron 提供跨平台支持，一套代码多端运行
React现代前端框架，组件化开发，状态管理清晰，适合构建复杂的看板 UI
Playwright微软出品，支持 Chromium/Firefox/WebKit，自动化能力强大且稳定
SQLite本地数据库，数据完全存储在用户电脑上。求职信息涉及个人隐私，不能上传云端
TypeScript类型安全，减少运行时错误，提升开发体验

和其他"自动投简历"工具有什么不同

市面上确实存在一些自动投简历的脚本或插件，但它们有几个明显的局限：

  • 只能在单一平台运行——某个招聘网站的自动化工具管不了公司官网、牛客、实习僧
  • 只解决"投递"这一个环节——不处理信息采集、不管理投递状态
  • 容易被封号——大量无差别投递很容易被平台识别

JobPilot 的思路不同：

  1. 全流程覆盖：从信息源 → 信息解析 → 投递管理 → 自动填表，一站式解决
  2. 跨平台：不限定在某一个招聘网站，任何有投递表单的页面都可以尝试自动填写
  3. 用户主动选择：不是无脑刷量，而是用户筛选后主动触发
  4. 隐私优先：所有数据本地存储，不经过任何第三方服务器

写在最后：不是技术多厉害，而是思维方式的转变

回头看这个项目，技术上并没有什么高深的东西——Electron、React、Playwright，都是成熟的开源工具。但我觉得它的价值在于用产品思维解决了一个真实的痛点。

求职本身就是一场信息战。谁能更快地获取信息、更高效地处理信息，谁就占据先机。JobPilot 帮我省下的不只是填表的时间，更是"不用在琐碎事务上消耗意志力"的心力。

而且，这种发现问题、拆解问题、用工具解决问题的能力，恰恰也是企业在招聘时希望看到的素质。所以某种意义上，做这个项目的过程，本身就是一次很好的求职准备。

如果你也在为秋招头疼，不妨试试 JobPilot。或者，你也可以从自己的痛点出发，造一个属于你自己的工具。

毕竟，最好的简历项目，就是解决你自己真正遇到的问题。
