个人网站配置总结

这份文件记录 idkyjun.me 当前个人网站的发布方式、域名状态和后续迁移到 Cloudflare DNS 时要注意的事情。它不是最终教程，而是一份可以继续更新的个人配置档案。

当前状态

  • 网站仓库：683280yj/683280yj.github.io
  • 发布方式：GitHub Pages，从 master 分支根目录发布。
  • 自定义域名：idkyjun.me，仓库保留 CNAME 文件。
  • 页面结构：中文首页 /，英文主页 /en/，中文分享中心 /share/。
  • HTTPS：GitHub Pages 侧暂时不能强制 HTTPS，原因是证书还没有由 GitHub Pages 签出；这和 DNS/代理链路有关。

已完成的配置

  1. 在 GitHub Pages 仓库中设置自定义域名 idkyjun.me。
  2. 保留 CNAME 文件，让 Pages 知道当前站点的正式域名。
  3. 重构首页：英文页作为实用个人主页，中文页作为文件总结与分享入口。
  4. 新增 /share/，把学习笔记、工具记录、比赛复盘和网站配置放到一个中文分享中心。
  5. 本地和线上确认 GitHub Pages 构建成功，新页面已经发布。

Cloudflare DNS 迁移建议

后续如果把 DNS 挂到 Cloudflare，建议让 Cloudflare 接管 DNS 和边缘 SSL，而不是迁移 Namecheap 送的 SSL 证书。GitHub Pages 常用记录可以先按下面准备：

A      @      185.199.108.153
A      @      185.199.109.153
A      @      185.199.110.153
A      @      185.199.111.153
CNAME  www    683280yj.github.io

迁移后再回 GitHub Pages 检查证书状态，等证书签出后再打开强制 HTTPS。
